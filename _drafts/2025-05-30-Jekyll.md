## excerpt (발췌문/요약)

### 용도

- 블로그 메인 페이지에서 포스트 미리보기로 표시
- SEO meta description으로 사용
- 소셜 미디어 공유 시 설명문으로 활용

### SEO 효과

```html
<!-- 자동 생성되는 meta 태그 -->
<meta name="description" content="Model Context Protocol과 Smithery를 통해 AI 앱이 외부 서비스와 연결하는 방법을 알아봅시다">
<meta property="og:description" content="Model Context Protocol과 Smithery를 통해...">
```

### 작성 요령

- 150-160자 내외 (구글 검색 결과 표시 길이)
- 핵심 키워드 포함
- 읽고 싶게 만드는 호기심 유발

## categories (카테고리)

### 용도

- 포스트 분류 및 그룹화
- 자동 카테고리 페이지 생성 (/categories/AI/, /categories/Development/)
- 사이드바 카테고리 위젯에 표시
- URL 구조에 반영 (/ai/development/2025/05/30/post-title/)

### 실제 효과

- 자동 페이지 생성:
    - /categories/ - 모든 카테고리 목록
    - /categories/ai/ - AI 카테고리 포스트들
    - /categories/development/ - Development 카테고리 포스트들
- 네비게이션:
    - Home > AI > Development > MCP와 Smithery 가이드
- 관련 포스트 표시:
    - 같은 카테고리의 다른 글들 자동 추천

### 설계 원칙

- 너무 세분화하지 말기 (5-10개 수준)
- 일관성 있는 명명 규칙
- 대소문자 통일 (보통 Title Case)

### Minimal Mistakes에서의 활용

```yml
## _config.yml 설정

# 카테고리 기반 permalink
permalink: /:categories/:year/:month/:day/:title/

# 카테고리 페이지 자동 생성
category_archive:
  type: liquid
  path: /categories/
```

```yml
## 사이드바 위젯

# _config.yml
sidebar:
  - title: "Categories"
    nav: "categories"
```

### SEO 효과

- 검색엔진 크롤링 최적화
- 사용자 탐색 편의성 향상
- 컨텐츠 발견 가능성 증대

### 용도

- 포스트 분류 및 그룹화
- 자동 태그 페이지 생성 (/tags/AI/, /tags/Development/)
